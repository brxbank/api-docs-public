# Cash-In (Receiving Payments)

Cash-In endpoints allow you to create PIX charges and generate QR codes for payers.

<Callout intent="info">
  Cash-In endpoints accept **API Key authentication** (`x-key-id` + `x-secret-key`) OR **Bearer Token**.
</Callout>

## Creating a Charge

Create a PIX charge using `POST /api/v1/pix/cash-in/sync`:

**Using API Key:**

```bash
curl -X POST https://api.brzip.com.br/api/v1/pix/cash-in/sync \
  -H "Content-Type: application/json" \
  -H "x-key-id: your-key-id" \
  -H "x-secret-key: your-secret-key" \
  -d '{
    "amount": "150.00",
    "reference": "order-12345",
    "description": "Payment for order #12345",
    "payer_name": "John Doe",
    "payer_document": "12345678901",
    "expiration_time": 3600
  }'
```

**Using Bearer Token:**

```bash
curl -X POST https://api.brzip.com.br/api/v1/pix/cash-in/sync \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer eyJhbGciOiJIUzI1NiIs..." \
  -d '{
    "amount": "150.00",
    "reference": "order-12345",
    "description": "Payment for order #12345",
    "payer_name": "John Doe",
    "payer_document": "12345678901",
    "expiration_time": 3600
  }'
```

**Response:**

```json
{
  "success": true,
  "transaction_id": "txn_f1e2d3c4-b5a6-7890-cdef-1234567890ab",
  "status": "pending",
  "qr_code": "00020126580014br.gov.bcb.pix0136...",
  "qr_code_text": "data:image/png;base64,iVBORw0KGgo...",
  "expires_at": "2026-02-23T22:10:00.000Z",
  "provider": "pix"
}
```

## Checking Charge Status

Query the status of a charge by its transaction ID:

```bash
curl -X GET https://api.brzip.com.br/api/v1/pix/cash-in/txn_abc123 \
  -H "x-key-id: your-key-id" \
  -H "x-secret-key: your-secret-key"
```

### Possible Statuses

| Status | Description |
|--------|-------------|
| `pending` | Charge created, awaiting payment |
| `paid` | Payment received and confirmed |
| `expired` | Charge expired without payment |
| `cancelled` | Charge was cancelled |
| `failed` | Charge processing failed |

## Cancelling a Charge

Cancel a pending charge:

```bash
curl -X POST https://api.brzip.com.br/api/v1/pix/cash-in/txn_abc123/cancel \
  -H "x-key-id: your-key-id" \
  -H "x-secret-key: your-secret-key"
```

<Callout intent="warning">
  Only charges with `pending` status can be cancelled.
</Callout>

## Advanced Features (CobV)

For charges with due dates (CobV — Charge with Due Date), you can include additional financial fields:

| Field | Description |
|-------|-------------|
| `due_date` | Due date (YYYY-MM-DD) |
| `penalty` | Late payment penalty (`mode`: 1=fixed, 2=percentage; `value`) |
| `interest` | Daily interest (`mode`: 1=BRL/day, 2=%/day, 3=%/month, 4=%/year; `value`) |
| `discount` | Early payment discount (`mode` + `value` + `fixed_dates[]`) |
| `rebate` | Rebate/abatement (`mode`: 1=fixed, 2=percentage; `value`) |
